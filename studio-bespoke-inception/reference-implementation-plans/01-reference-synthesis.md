## Cross-reference conclusion

The corpus does not justify making Studio Bespoke more kinetic. It validates a narrower direction:

1. Keep one stable visual idea while meaning changes around it.
2. Use one authored pin for the flagship proof, not a page full of pins.
3. Let information-heavy chapters become calmer than identity or threshold chapters.
4. Reuse project imagery through deliberate crop changes instead of introducing unrelated media.
5. Make navigation and inquiry remain spatially stable while the narrative moves.
6. Prefer reversible two-dimensional transforms—translation, scale, clipping and opacity—to WebGL, 3D rotation or autonomous spectacle.

This supports the existing Ritual Plan. It does not replace it.

## Per-reference brand mapping

### R1 — Sakazuki

**Observed reference behaviour**

- A small central angular mask expands into the hero.
- A stable three-zone editorial grid changes through three numbered states.
- One central artwork persists while large type passes around it.
- Repeated notched panels turn one motif into a family of transitions.
- Motion density falls during philosophy and FAQ reading scenes.

**Recommended Studio Bespoke adaptation**

- Use the central expansion only as validation for the already-approved Mira arch aperture.
- Adapt the stable numbered editorial composition into the Rituals chapter, but use a sticky image rail and normally scrolling captions rather than another heavy GSAP pin.
- Preserve the principle of reduced motion during proof explanation and inquiry.

**Reject**

- Crimson/black identity theatre, giant kinetic member names, geometric shards, animated footer density, background video and repeated panel covers.
- Reusing the arch as a decorative notch on every section. The arch belongs primarily to the intervention proof.

**Confidence:** High for the aperture and panel family; medium for the exact trigger of the philosophy-card sequence.

### R2 — Adcker / adkr

**Observed reference behaviour**

- The opening reveals typography through horizontal slices.
- Images function as glyphs inside oversized type.
- Service descriptions expand in document flow after pointer activity.
- A five-image row isolates the active image by desaturating siblings.
- A simple dark cover introduces contact; the final wordmark becomes a full-screen mass.

**Recommended Studio Bespoke adaptation**

- Use the disclosure principle for optional supporting information, never for primary proof.
- Retain the idea that active media can be distinguished without enlarging it; if used, apply a quiet opacity change to supporting-work previews with full keyboard and touch equivalents.
- Let inquiry arrive through a clear register change, but keep Studio Bespoke's mineral restraint.

**Reject**

- Percentage loader, fashion showreel, type-as-image spectacle, giant services typography, custom cursor and giant closing wordmark.
- Hover as the only way to reveal a project decision.

**Confidence:** High for the visual states; medium for click-versus-hover disclosure triggering.

### R3 — French interior-architecture portfolio

**Observed reference behaviour**

- A reversible pinned mosaic changes project imagery and a central information card.
- A small portrait image expands under scroll into a full-frame Process scene, and contracts on reverse.
- Seven numbered process states use a consistent split image/information composition.
- Project thumbnails later resolve into a project grid; project pages use art-directed image groupings rather than generic masonry.

**Recommended Studio Bespoke adaptation**

- Borrow the logic of a stable process template, reduced from seven stages to Studio Bespoke's three verified delivery stages.
- Use large numbers as orientation, not decorative counters.
- Adapt art-directed image groupings for Zulal and Desert Leaf, with one dominant image and one supporting detail per case.
- Keep route-cover and project-index mechanics for a later portfolio-detail phase, not the homepage implementation.

**Reject**

- The opening project carousel, project-specific colour cards, full grid/filter system and a second full-frame expansion on the homepage.
- Calling still photography “proof” without accompanying decisions.

**Confidence:** High. Reversals and pointer interactions provide the strongest behavioural evidence in the corpus.

### R4 — Son Daven

**Observed reference behaviour**

- Hero media shrinks from full bleed into an inset object while a new visual world grows around it.
- Sections retain a surface, image or illustration while the next chapter enters.
- A horizontal card procession maps vertical scroll to lateral movement.
- Credentials and financial information use large numerals with disciplined supporting copy.
- Conversion, FAQ and footer remain connected by one illustration language.

**Recommended Studio Bespoke adaptation**

- Apply the continuity principle to the opening-to-inherited-plan handoff: one real blueprint plane, no panel edge and no duplicated unrelated drawing.
- Use the horizontal-procession idea only as a compositional reference for the Rituals sequence; implement a vertical sticky editorial rail to avoid mobile and accessibility costs.
- Use large stage numbers and short factual labels in Process & Trust.
- Carry one quiet rule/caption cadence from Process into Inquiry instead of introducing a new footer spectacle.

**Reject**

- Long multi-pin storytelling, investment-style statistics, illustrated landscapes, ticker headlines, card overlap density and an extended horizontal scroller.
- Any “before” transformation implied by a visual effect when no authentic before asset exists.

**Confidence:** High for continuity and visual movement; medium for exact pin/sticky implementation.

### R5 — OBYS

**Observed reference behaviour**

- One circular aperture evolves from loader to project selector to route-transition anchor.
- Project metadata remains fixed while media moves vertically in a separate field.
- Active selection is communicated through position, colour and typography rather than larger UI chrome.
- Project galleries appear cyclic, although infinite looping is not conclusively proven.

**Recommended Studio Bespoke adaptation**

- Validate the arch as the singular proof anchor: it opens, carries the Mira room, and then yields.
- Use fixed-information/moving-media separation in a lighter form for Rituals: captions stay readable while Mira details change.
- Keep project metadata peripheral and precise.

**Reject**

- Infinite work index, cyclic galleries, monochrome agency UI, persistent aperture across every chapter and route-progress line.
- Turning the homepage into a portfolio browser before the renovation argument is established.

**Confidence:** Medium-high. The visible states are clear, but scroll input, looping and clicks are partly inferred and no reverse scroll was shown.

### R6 — Fluid Glass

**Observed reference behaviour**

- Architectural imagery receives more space than explanatory copy.
- A stable navigation device remains available through long pages and route changes.
- Project rows reveal floating image previews on hover.
- Technical configurations use large explanatory copy with precise diagrams.
- A restrained approach panel leads into a photographic footer.

**Recommended Studio Bespoke adaptation**

- Keep a restrained persistent header and a quiet always-reachable inquiry link.
- Use thin rules, short labels and image-safe text placement in supporting work and process.
- Borrow the project-row preview only for future work-index routes; on the homepage, show the two supporting cases directly.
- Let the approach chapter combine emotional trust with exact delivery information.

**Reject**

- Permanent bottom navigation capsule, autoplay showroom media, product catalogue tabs, route panels on the homepage and a photographic footer that competes with inquiry.

**Confidence:** High for interaction and repeated route grammar; medium for showroom media type and any sticky product hero.
